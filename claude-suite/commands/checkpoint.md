Produce a concise checkpoint of the current session. No skill — keep it short:

1. **Changed this session** — what was actually edited or created, by file.
2. **Verified** — what has been confirmed working (gates run, tests passed, builds green).
3. **Remaining** — what is still open or untested.
4. **Next action** — the single next thing to do.

A few bullets per section. Do not pad; if a section is empty, say so.
